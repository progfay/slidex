# slidex — スライド生成規約

Claude Code は **slides/ 以下の生成・編集**が主な仕事で、engine/ と design-system/ は明示的な指示がない限り変更しない。
対応ブラウザは最新の Chromium 系のみ(詳細は [README.md](./README.md#動作要件))。

変更は意味のあるまとまりごとに git commit する(未コミットのまま溜めない)。

## スライドを作る流れ

デッキは次の5ステップで作る。

ステップは一方通行ではない。前のステップに戻って直してよい。
**正は常に一番下流のドキュメント**であり、上流ファイルへの逆同期は任意。

### 1. ブレインダンプを書く — `braindump.md`

ユーザーに書かせるのではなく、まず AI が対話する。話したい理由・想定聴衆・
伝えたいエピソード・持ち帰ってほしいことを掘り下げ、**コンセプトの輪郭が
見えてくるまで十分に議論する**(聞き役に徹し、早々にまとめない)。

出てきた要点・語り・素材を `braindump.md` に書き出す(リポジトリルート、
構成は気にせず思いつくまま)。

### 2. コンセプトをまとめる — `concept.md`

`braindump.md` から核を抽出し `concept.md` に書く:

- 一言メッセージ(聞き手に持ち帰ってほしいことを1文で)
- 想定聴衆
- 目的(伝える・説得する・行動を促す、など)
- 根拠・柱(2〜3個)

braindump.md を材料に、どうまとめるかはユーザーと対話しながら決める。

ここがぶれるとスライド構成全体がぶれるので、確定したら
ユーザーに確認を取り、`manifest.json` の `title` をデッキ名に書き換える。

### 3. アウトラインを作る — `outline.md`

`concept.md` を軸に、スライド単位の構成へ展開する。1スライド = 1セクション
(`##`)とし、セクションごとに書くのは:

- そのスライドの一言メッセージ
- スライドに載せる要点(見出し案・箇条書き案)
- 話す内容の語り(そのまま発表者ノートの素材になる)

`braindump.md` の素材の取捨選択・並べ替えはここで行う。判断が割れやすいので、
外した素材や構成の順序案は理由を添えてユーザーと相談する。確定したらユーザーに確認を取る(ここから先は直すコストが上がる)。

**デッキの最後は必ず「まとめ」セクションにする**(`concept.md` の一言メッセージ・
根拠/柱をベースに箇条書きでまとめる)。

### 4. HTML スライドに書き起こす

`outline.md` の 1セクション = 1スライドで `slides/NN-slug.html` に変換し、
`manifest.json` の `slides` 配列を並び順どおり更新する。

- 画面に載せるのは要点だけ。**アウトラインの文章をそのまま流し込まない**
  (「見出し + 箇条書き4項目」の上限目安はここで守る)
- 語りは `<aside class="notes">` に発表者ノートとして転記する
- スライドごとに適切なレイアウト(`layout-*`)を選ぶ

### 5. リッチ化する

素の書き起こしを、HTML の柔軟性を活かして分かりやすく・見やすくする。

- 箇条書きの図解化(`layout-freeform` での図・対比・タイムライン)
- コードハイライト、`.accent` での強調、CSS による演出
- ただし規約は維持する: 生の値ではなくデザイントークンを使い、
  1スライドの情報量は増やさない。**リッチ化 = 装飾の追加ではなく伝達効率の改善**

## スライドHTML

HTML構造の契約・レイアウト一覧(`layout-*`)・デザイントークンは skill
